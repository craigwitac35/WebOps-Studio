Guardian 4 Heroes

🧭 Goal

Guardian 4 Heroes is a nonprofit providing equine therapy to veterans and first responders. They had no web presence whatsoever — no site, no donation pathway, nothing. The goal was to build a complete, mission-aligned site from scratch that could communicate their story, build trust with visitors, and drive donations.

⚠️ Challenges

  • Zero existing design assets, brand guidelines, or content framework to start from
  • Needed to balance emotional storytelling with functional conversion paths (donations)
  • Had to work on a nonprofit budget — no bloat, no expensive integrations
  • Required full mobile responsiveness from day one, as most visitors come from phones

🔧 Solution

Built entirely from scratch using HTML5, CSS3, and vanilla JavaScript — no frameworks, no templates. Established a visual identity that reflected warmth, healing, and trust. Prioritized a clear donation pathway with tiered buttons and a custom amount option.

  • Designed and built a full hero section with a silhouette-style background evoking connection and calm
  • Built a fully working contact form wired to email via Formspree
  • Integrated a tiered donation section with PayPal links
  • Custom mobile nav with smooth transitions and accessible markup
  • High-contrast, readable type system — accessible to all age groups

🎯 Result

The organization launched with a site that matched their identity and gave them a credible, functional hub for donors, volunteers, and the community.

"Craig went above and beyond. He didn't just build us a website — he brought our mission to life. We get compliments on it constantly."
  • Clear, working donation pathway live from day one
  • Contact form receiving real inquiries
  • Fully responsive — performs on desktop and mobile equally
  • Hosted on GitHub Pages (zero ongoing hosting cost for the nonprofit)

🔁 What I'd Improve Next

Add a proper CMS so the team can update event dates and news without touching code. Also worth A/B testing different donation tier amounts to optimize conversion rate.

Construction Business Site

🧭 Goal

A local Minnesota construction company needed a professional web presence to compete with larger contractors. The site needs to communicate credibility, showcase past work, and make it dead simple to request a quote or call.

⚠️ Challenges

  • Construction clients often judge on first impression — the site has to look established and trustworthy
  • Large image galleries need to load fast on mobile (most quote requests come from phones on job sites)
  • Simple enough for the owner to hand content to without needing a developer

🔧 Solution

Building a clean, dark-toned 5-page site with a filterable project gallery, a structured services section with clear scope descriptions, and a prominent quote request form. Mobile-first layout optimized for large images with lazy loading.

  • Services page with scope cards: residential, commercial, renovation
  • Project gallery with category filters (no JS library — custom built)
  • Quote request form with job type dropdown and message field
  • Click-to-call CTA persistent in mobile header

🎯 Result

Currently in development. Expected launch within the next 4–6 weeks. Screenshots and live link will be added here on completion.

Local Cafe Website — Concept

🧭 Goal

A concept build demonstrating how a small local cafe could have a premium, custom web presence without paying for an agency. The goal is a warm, atmospheric site that makes you want to walk through the door — with a menu, events, and a simple reservation flow.

⚠️ Challenges

  • Small business sites often look cheap — the challenge is premium feel on a tight scope
  • Menu content changes frequently — needed a structure easy to update with basic HTML knowledge
  • Warm, inviting aesthetic without using off-brand stock photography

🔧 Solution

Designing a warm-toned, minimal 4-page site: Home, Menu, Events, and Contact. The menu is structured as clean HTML tables — easy to edit in any text editor. Events section uses a simple card grid. Reservation mock-up uses a form with date/time inputs.

  • Custom color palette — warm cream, deep espresso, gold accent
  • Menu page: categorized items, clean pricing, no bloat
  • Events grid with date, title, and description cards
  • Reservation form mock-up with date/time picker inputs

🎯 Result

Concept phase — a live demo is planned. This project exists to demonstrate the range of aesthetic systems I can build: not everything is dark and minimal. A good developer adapts to what the brand actually needs.

🔁 What I'd Improve Next

Integrate a real booking API (like OpenTable or a simple Calendly embed) and add a subtle entrance animation on the menu items to give it a boutique feel.
